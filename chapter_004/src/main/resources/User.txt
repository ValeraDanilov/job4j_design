Подсчёт идёт из расчёта 64 разрядной систем.

 - 1 объект.
 User {
 Заголовок -> 16 байт.
 String name; -> занимает 56 байт (сам объект string 32 и массив char[] в String 24 ,байта).
 int age; -> занимает 4 байта.
 В общей сумме объект User будет занимать 76 байт.
 }


 - 2 объект.
  User {
  Заголовок -> 16 байт.g
  String name; -> занимает 56 байт (сам объект string 32 и массив char[] в String 24 ,байта).
  int age; -> занимает 4 байта.
  int id;  -> занимает 4 байта.
  В общей сумме объект User будет занимать 80 байт.
  }

В Edit Configurations  можно установить начальный и максимальный размер heap через (-Xms -Xmx) таким образом можно добиться вызова сборщика
мусора если установить минимальный размер памяти и заполнить её объектами через цикл, как это указанно в примере.
